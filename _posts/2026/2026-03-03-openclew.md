[OpenClaw](https://openclaw.ai/) (formerly Clawdbot, Moltbot) is a free, open-source autonomous AI agent framework that lets you control your computer and automate tasks using natural language, all running locally on your machine. ([OpenClaw introduction](https://mp.weixin.qq.com/s/JwlZimUm8RwtntnZFHosjQ))

* **How it works**: It uses an external LLM (like GPT, Claude, Gemini) for reasoning, but runs and executes commands locally on your Windows/macOS/Linux device.
  1. **Interface**: Controlled via chat apps (Telegram, WhatsApp, Discord, Slack).
  2. OpenClaw’s **agent core** processes it, deciding on actions.
  3. **Commands** : exec, read, process, session history.
  4. It acts and reports back through the same channel.

* **Skills ecosystem** : pre-written token, to describe each task.
  * [awesome openclew skills](https://github.com/VoltAgent/awesome-openclaw-skills), [clawhub](https://clawhub.ai/)
* ***Problems***:
  * skills could have **safety issues** : Because it can access emails, login tokens, files, command execution, and other sensitive systems, misconfiguration or unvetted community extensions can expose serious risks. Security audits and careful configuration are essential.
  * **context collapse** (上下文灾难) where accumulating conversation history and irrelevant information degrade reasoning quality, consistency, and decision-making accuracy over time.


Related Works:
* [ZeroClaw](https://github.com/zeroclaw-labs/zeroclaw) : a lightweight, drop-in replacement for OpenClaw written in Rust.
* [AReaL: A Large-Scale Asynchronous Reinforcement Learning System](https://github.com/inclusionAI/AReaL) training the agent’s “decision-making and skill usage” via RL.

### Make My Claw
<p></p>

* LLM using [Qwen3.5](https://github.com/QwenLM/Qwen3.5):
  * [Ollama on Jetson AGX Orin](https://www.jetson-ai-lab.com/tutorials/ollama/), [Ollama to OpenClaw](https://docs.openclaw.ai/providers/ollama)
  * ✅ [Jetson AI Lab Qwen3.5 35B-A3B (MoE)](https://www.jetson-ai-lab.com/models/qwen3-5-35b-a3b/), [vLLM to OpenClaw](https://docs.openclaw.ai/providers/vllm)
  (saving the model to [external sd card](https://oneuptime.com/blog/post/2026-02-08-how-to-move-dockers-storage-location-to-an-external-drive/view))

```
sudo docker run -it --rm --runtime=nvidia --network host \
  -v /mnt/sdcard/vllm_models:/vllm_cache \
  -e TRANSFORMERS_CACHE=/vllm_cache \
  -e HF_HUB_CACHE=/vllm_cache/hub \
  -e HF_HOME=/vllm_cache \
  -e VLLM_MARLIN_USE_ATOMIC_ADD=1 \
  ghcr.io/nvidia-ai-iot/vllm:latest-jetson-orin \
  vllm serve Kbenkhaled/Qwen3.5-35B-A3B-quantized.w4a16 \
    --gpu-memory-utilization 0.8 --enable-prefix-caching \
    --api-key BijPnpytnrtqqXJuippPmWfm \
    --reasoning-parser qwen3 \
    --speculative-config '{"method": "mtp", "num_speculative_tokens": 4}' \
    --enable-auto-tool-choice --tool-call-parser qwen3_coder
```
<p></p>

* OpenClaw Agent - using [ROC-RK3588S-PC](https://wiki.t-firefly.com/en/ROC-RK3588S-PC/upgrade_firmware.html) with linux system.
  * [Install OpenClaw](https://docs.openclaw.ai/install): `curl -fsSL https://openclaw.ai/install.sh | bash`
  * `openclaw configure` configuration in "~/.openclaw/openclaw.json"
  * `openclaw reset` to start over
